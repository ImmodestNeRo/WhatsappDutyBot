# DutyBot

Бот для автоматичного керування чергуваннями у WhatsApp-групі. Веде чергу, призначає чергових щоранку, надсилає нагадування та фіксує порушників.

## Як працює

Кожного робочого дня о 08:00 бот закриває попередній день (ротація + фіксація порушників), потім призначає нового чергового і надсилає оголошення в групу. Черговий має підтвердити командою `/done` до наступного ранку (24 години). Якщо підтвердження не надійшло — бот записує порушення.

**Розклад (налаштовується):**
- `08:00` — закриття попереднього чергування + призначення нового
- `14:00` / `17:30` — нагадування, якщо не підтверджено
- Неділя — вихідний, бот надсилає повідомлення про вихідний і не призначає чергового

## Штрафна система

Якщо черговий не підтвердив чергування до наступного ранку — він отримує штраф. Штрафи зберігаються у `pending_penalties` і застосовуються автоматично в кінці циклу.

**Цикл** — один повний оберт черги (кожен учасник почергував по одному разу). Після завершення циклу формується нова черга, де кожен порушник отримує **додатковий слот** рівно поряд зі своїм основним (2 дні поспіль на наступному циклі). Кількість додаткових слотів дорівнює кількості пропусків.

Штрафи видно у `/list` і `/longlist` — слоти відмічені як `⚠️ штрафний`. Якщо є незастосовані штрафи (наступний цикл ще не почався) — вони показані внизу списку.

## Команди

### Публічні

| Команда | Опис |
|---------|------|
| `/add` | Додати себе до черги |
| `/done` | Підтвердити чергування |
| `/list` | Черга на 10 найближчих днів |
| `/rat [номер або @згадка]` | "Донести" на вказаного учасника (або на випадкового) |
| `/swap` | "Обмінятись" чергуванням (спойлер: ні) |
| `/help` | Довідка по командах |

### Адміністраторські

| Команда | Опис |
|---------|------|
| `/add <номер або @згадка>` | Додати когось (можна кілька через пробіл або @згадки) |
| `/remove <номер або @згадка>` | Видалити з черги |
| `/remove-q` | Очистити всю чергу |
| `/remove-g` | Очистити список порушників |
| `/guilty` | Список порушників з лічильником пропусків |
| `/longlist` | Повна черга |
| `/trigger` | Надіслати нагадування черговому вручну |
| `/skip` | Пропустити поточного чергового (без штрафу) — переміщує в кінець черги |
| `/dogana <номер або @згадка>` | Записати ручне порушення (штраф) |
| `/pardon <номер або @згадка>` | Зняти одне порушення |
| `/admins-list` | Показати адмінів |
| `/bind_group` | Прив'язати бота до групи |

## Конфігурація

Всі налаштування задаються через змінні оточення (файл `.env` або панель хостингу).

| Змінна | Опис | За замовчуванням |
|--------|------|------------------|
| `GROUP_JID` | JID групи для авто-прив'язки | — |
| `ADMIN_PHONES` | Телефони адмінів через кому (без `+`) | — |
| `TIMEZONE` | Часовий пояс | `Europe/Kyiv` |
| `SCHEDULE_MORNING` | Ротація + призначення чергового | `08:00` |
| `SCHEDULE_REMINDER_1` | Перше нагадування | `14:00` |
| `SCHEDULE_REMINDER_2` | Друге нагадування | `17:30` |
| `QUEUE_ALWAYS_LAST` | Номер, який завжди останній у черзі | — |
| `RATE_LIMIT_CALLS` | Макс. команд на користувача за вікно | `5` |
| `RATE_LIMIT_WINDOW` | Вікно ліміту (секунди) | `60` |
| `DATA_DIR` | Директорія для даних | `./data` |
| `HISTORY_SYNC_GRACE` | Ігнорування старих повідомлень при старті (сек.) | `60` |
| `LOG_LEVEL` | Рівень логування | `INFO` |

## Деплой на Railway

### 1. Створити сервіс

Railway Dashboard → New Project → Deploy from GitHub repo → обрати репозиторій.

### 2. Налаштувати Volume

Settings → Volumes → Add Volume → Mount Path: `/app/data`

Volume зберігає сесію WhatsApp і стан черги між редеплоями.

### 3. Додати змінні

Settings → Variables — додати всі потрібні змінні з таблиці вище. Обов'язково:
- `DATA_DIR` = `/app/data`
- `GROUP_JID` — JID вашої групи
- `ADMIN_PHONES` — телефони адмінів

### 4. Авторизація WhatsApp

Після першого деплою бот згенерує QR-код. Відкрийте Deploy Logs у Railway і відскануйте QR через WhatsApp → Налаштування → Підключені пристрої → Підключити пристрій.

Після сканування сесія зберігається у Volume — наступні редеплої не потребують повторної авторизації.

## Локальний запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заповнити змінні
python main.py          # відсканувати QR при першому запуску
```

### Docker Compose

```bash
cp .env.example .env
docker-compose up -d
docker logs -f duty-bot
```

## Поради щодо WhatsApp

**Додати бота до контактів** — всі учасники групи мають зберегти номер бота у контактах. WhatsApp блокує акаунти, що масово пишуть незнайомцям.

**LID** — WhatsApp у групах замінює номери на технічні ідентифікатори. Бот розпізнає їх автоматично. Якщо не вдається — учасник може додати себе вручну: `/add 380XXXXXXXXX`.

## Структура проєкту

```
main.py                  — точка входу, auto-reconnect цикл
config.py                — конфігурація з .env (всі змінні)
start.sh                 — entrypoint: seed черги в Volume при першому деплої
bot_state.seed.json      — початковий стан черги для першого деплою
Dockerfile               — збірка образу
docker-compose.yml       — локальний запуск з Volume
.env                     — змінні оточення (не комітити!)
requirements.txt         — залежності Python

services/
  whatsapp.py            — WhatsApp-клієнт, dispatch команд, відправка повідомлень
  duty.py                — бізнес-логіка: черга, призначення, ротація, порушники
  scheduler.py           — cron-задачі (ранок/нагадування) + catch-up при старті
  storage.py             — атомарний JSON-стейт з блокуванням і fsync
  messages.py            — всі тексти повідомлень бота
  utils.py               — логування, retry-декоратор, rate limiter

data/                    — директорія даних (не комітити!)
  session.db             — WhatsApp-сесія
  bot_state.json         — поточний стан черги і чергувань
  logs/bot.log           — ротаційні логи
```
