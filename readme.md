# 🤖 DutyBot — Платформа для керування чергуваннями у WhatsApp

Цей бот автоматизує процес призначення чергових у WhatsApp-групах. Він веде чергу, надсилає ранкові оголошення, нагадування протягом дня та фіксує тих, хто не виконав своє завдання.

## 🚀 Основні можливості

- **Автоматичне призначення**: Щоранку бот оголошує нового чергового з черги.
- **Розумна черга**: Пропускає неділі (вихідний) та автоматично розраховує дати наступних чергувань.
- **Система нагадувань**: Якщо черговий не підтвердив виконання завдання, бот надішле нагадування (вдень та ввечері).
- **Список "шкодників"**: Автоматично фіксує користувачів, які пропустили своє чергування.
- **Підтримка LID**: Автоматично розпізнає контакти навіть якщо WhatsApp приховує їхні справжні номери (через LID).
- **Docker-ready**: Легке розгортання через Docker та Docker Compose.

## 🛠 Налаштування та запуск

### 1. Підготовка оточення
Створіть файл `.env` на основі прикладу:
```bash
cp .env.example .env
```
Відредагуйте `.env`, вказавши часовий пояс та (опціонально) JID групи.

### 2. Запуск (через Docker Compose)
```bash
docker-compose up -d
```

### 3. Авторизація
Після першого запуску відкрийте логи бота (`docker logs -f dutybot`), щоб побачити **QR-код**. Відскануйте його через WhatsApp (Прив'язані пристрої) на вашому телефоні.

## 📱 Команди бота

Всі команди потрібно надсилати у групу, де працює бот.

| Команда | Опис |
| :--- | :--- |
| `/bind_group` | Прив'язати бота до поточної групи (якщо не вказано `GROUP_JID` в `.env`). |
| `/add [номер/згадка]` | Додати користувача до черги. Якщо без аргументів — додає вас. |
| `/remove [номер/згадка]` | Видалити користувача з черги. |
| `/list` | Показати поточну чергу з датами на найближчі 10 днів. |
| `/guilty` | Показати список тих, хто пропустив чергування. |
| `/done` | Підтвердити виконання чергування за сьогодні. |

## ⚙️ Конфігурація (.env)

| Змінна | Опис | За замовчуванням |
| :--- | :--- | :--- |
| `TIMEZONE` | Часовий пояс для планувальника. | `Europe/Kyiv` |
| `SCHEDULE_MORNING` | Час ранкового оголошення. | `08:00` |
| `SCHEDULE_REMINDER_1` | Перше нагадування. | `14:00` |
| `SCHEDULE_REMINDER_2` | Друге нагадування. | `17:30` |
| `SCHEDULE_END_OF_DAY` | Час ротації та перевірки "шкодників". | `23:59` |
| `DATA_DIR` | Шлях до папки з даними (стан, логи). | `./data` |

## 🏗 Структура проекту

- `main.py` — точка входу.
- `services/whatsapp.py` — логіка взаємодії з WhatsApp (neonize).
- `services/duty.py` — бізнес-логіка чергувань та черги.
- `services/scheduler.py` — планувальник завдань.
- `services/messages.py` — всі текстові повідомлення бота (можна змінювати тут).
- `data/bot_state.json` — збережений стан черги та налаштувань.

## 👨‍💻 Розробка

Для локального запуску без Docker:
1. Встановіть залежності: `pip install -r requirements.txt`
2. Переконайтеся, що у вас встановлено `sqlite3` та залежності для `neonize`.
3. Запустіть: `python main.py`
