# DutyBot — Бот для керування чергуваннями у WhatsApp

Автоматизує призначення чергових у WhatsApp-групі: веде чергу, надсилає ранкові оголошення, нагадування та фіксує порушників.

## Команди бота

Надсилати у групу, де працює бот.

| Команда | Опис |
| :--- | :--- |
| `/bind_group` | Прив'язати бота до поточної групи (Адмін) |
| `/add` | Додати себе до черги (Публічна) |
| `/add [номер або @згадка]` | Додати одного або кількох учасників (Адмін) |
| `/remove [номер або @згадка]` | Видалити одного або кількох учасників (Адмін) |
| `/remove-q` | Очистити всю чергу (Адмін) |
| `/remove-g` | Очистити список шкодників (Адмін) |
| `/list` | Черга на найближчі 10 днів з датами (Публічна) |
| `/longlist` | Повна черга (Адмін) |
| `/guilty` | Список тих, хто пропустив чергування (Публічна) |
| `/done` | Підтвердити виконання чергування (Публічна) |
| `/admins-list` | Список налаштованих адміністраторів (Адмін) |
| `/help` | Довідка по командах (Публічна) |

## Конфігурація (.env)

| Змінна | Опис | За замовчуванням |
| :--- | :--- | :--- |
| `ADMIN_PHONES` | Список телефонів адмінів через кому (без `+`) | _(порожньо)_ |
| `RATE_LIMIT_CALLS` | Макс. кількість команд за вікно | `5` |
| `RATE_LIMIT_WINDOW` | Вікно лімітування (в секундах) | `60` |
| `TIMEZONE` | Часовий пояс | `Europe/Kyiv` |
| `GROUP_JID` | JID групи для авто-прив'язки при старті | _(порожньо)_ |
| `SCHEDULE_MORNING` | Час ранкового оголошення | `08:00` |
| `SCHEDULE_REMINDER_1` | Перше нагадування | `14:00` |
| `SCHEDULE_REMINDER_2` | Друге нагадування | `17:30` |
| `SCHEDULE_END_OF_DAY` | Ротація та перевірка порушників | `23:59` |
| `DATA_DIR` | Папка для даних (session.db, логи, стан) | `./data` |
| `HISTORY_SYNC_GRACE` | Секунд ігнорування старих повідомлень при старті | `60` |
| `LOG_LEVEL` | Рівень логування (`DEBUG`, `INFO`, `WARNING`) | `INFO` |

## Деплой

### Важливо: авторизація перед деплоєм

WhatsApp-сесія зберігається у `data/session.db`. **Деплоїти потрібно з вже активною сесією** — інакше бот не матиме до кого підключитися.

**Правильний порядок:**
1. Запустити бота локально: `python main.py`
2. Відсканувати QR-код через WhatsApp → Прив'язані пристрої
3. Переконатися що бот підключився (з'явиться лог `WhatsApp connected successfully`)
4. Зупинити бота
5. Скопіювати `data/session.db` на сервер / у volume
6. Деплоїти

### Railway

1. Завантажити `data/session.db` в Railway Volume (mount path `/app/data`)
2. Додати всі змінні з `.env` у `Settings → Variables`
3. Запушити код — Railway збере і задеплоїть автоматично

### VPS / Docker

```bash
git clone <repo>
cd DutyBot
cp .env.example .env        # заповнити змінні
mkdir -p data
cp /звідки/session.db data/ # скопіювати сесію
docker-compose up -d
```

## Важливі поради щодо WhatsApp

### Додати бота до контактів

**Обов'язково** — всі учасники групи мають додати номер бота до своїх контактів на телефоні. WhatsApp агресивніше банить повідомлення від незнайомих номерів. Якщо бот буде надсилати багато повідомлень незнайомцям — акаунт заблокують.

### Перший запуск у групі

Після підключення:
1. Надіслати `/bind_group` у групу (або вказати `GROUP_JID` у `.env`)
2. Додати всіх учасників через `/add` або попросити їх самих написати `/add`

### LID (прихований номер)

WhatsApp у групах замінює реальні номери на технічні ідентифікатори (LID). Бот автоматично їх розпізнає через neonize. Якщо LID не вдається розв'язати (рідкісний випадок) — учасник може додати себе вручну через `/add 380XXXXXXXXX`.

### Уникнення бану

- Не надсилати більше кількох повідомлень за хвилину
- Використовувати реальний номер (не віртуальний)
- Акаунт має бути активним хоча б кілька тижнів до запуску бота
- Усі учасники групи мають мати бота в контактах

## Структура проєкту

```
main.py                  — точка входу, auto-reconnect цикл
config.py                — конфігурація з .env
services/
  whatsapp.py            — підключення до WhatsApp, обробка команд
  duty.py                — бізнес-логіка черги та чергувань
  scheduler.py           — cron-джоби + catch-up при старті
  storage.py             — атомарний JSON-стейт з блокуванням
  messages.py            — всі тексти повідомлень
  utils.py               — логування, retry-декоратор
data/
  session.db             — WhatsApp-сесія (не комітити!)
  bot_state.json         — стан черги, поточний черговий
  logs/bot.log           — ротаційні логи
```
